Lesson 1.1: Basics of the Web and HTML

What is the web?

The web is a bunch of computers that communicate with each other.

How does the web work?

The web has four main components:

  1. You: You want to visit a web page
  2. Your computer plus a Browser: Your browser makes a request via the internet
  3. The internet: Sends an HTTP request to a server
  4. Server: Locates HTML document and sends it to your computer where your web browser interprets and displays it on your screen

What is a web page?

A web page is a document written in a language called HTML. Web browsers read these documents, interpret and display them.

What is HTML?

HyperText Markup Language is the heart of the web.

HTML is made up of:

  • Text content (What you see)
  • Markup (What the text looks like)
  • References to other documents (eg: images, videos, etc...)
  • Links to other pages

Basic HTML vocabulary you should know:

Element:

A structure in HTML that consists of an opening tag, content, and a closing tag.

Elements can be either inline (effecting just the text enclosed by tags), or block (block elements have an invisible box around them with height and width that separates them from any other element). While most Elements have both an opening and closing tag, some Elements use only one tag and are referred to as void.

Tags:

Tags mark the beginning and end of an Element. They are made of a less-than symbol, followed by the Element's name, followed by a greater-than symbol.

Attributes and values:

Attributes follow tags and point to a value. Examples are images and links to other web pages

A Couple Other Points to Remember:

Computers are stupid

Why? Because they interpret instruction literally. If you use the wrong Element name in your opening tag or make a typo, a computer cannot "know" what you intended. A program will only work if the instructions are clear and the grammar correct.

There Is Far Too Much to Remember

Don't try to memorize every single fact. Instead, concentrate on understanding how programming languages work, and rely on the internet to remind you of an Element's name or a particular structure.

Understanding the Box Model

HTML boxes contain the following:

  1. Margin
  2. Border
  3. Padding
  4. Content

You can see an element's box in Dev Tools by clicking the Computed tab

At the center of this box is the Content.

  • The image or text that appears on the website
  • Consider the content the nucleus: It sits at the center and the surrounding layers protect it, so that other items don't overlap it or sit too close, making the website visually unappealing.

Next is the Padding.

  • The Padding clears the area around the Content.
  • It is affected by the background-color of the box
  • Think of it as the space inside the box that protects the Content directly.

Next is the Border.

  • The Border goes around both the Padding and the Content
  • It is inherited from the color property of the box.

Last, the out layer is the Margin.

  • The Margin clears an area around the Border
  • It has no background-color. It is completely transparent.
  • Think of it as the space between boxes

Calculating Box Size

Default = Sum of Border, Padding and Content

To make the calculation easier, we put the following code into our CSS file:

  1. * {
  2. box-sizing: border-box;
  3. }

Now, the calculation of the box includes both the border and the padding, so that it's easier to make layouts.

You just set the width of the overall box, and no matter how you tweak the padding and border layers, the size of the box will always stay the same.

The CSS must include browser-specific prefixes:

  • -webkit-box-sizing: border-box;
  • -moz-box-sizing: border-box;
  • -ms-box-sizing: border-box
  • box-sizing: border-box

You can set the size of your box layers in pixels or in percentages:

  • % changes the box as the screen size changes
  • Pixel sets the image to one size, regardless of device. This can lead to a bad user experience

You can set max-width, so that an element can get smaller, but not larger than a set value. Max-width won't change layout.

Lesson 2.1: Introduction to Serious Programming

What is Computer Science?

Computer Science is about how to solve problems by breaking them into smaller pieces and then mechanically and precisely describing the sequence of steps that you would use to solve each piece... and those steps can be executed by a computer.

What is a computer?

A universal machine that we can program to do any computation. Its power lies in doing things very fast: billions of instructions per second.

What is a program?

A very precise sequence of steps that tells the computer what to do. Anything we can write a program for, we can make a computer do.

To learn computing, we need to learn a programming language.

What is a programing language?

A vocabulary and set of grammatical rules for instructing a computer to perform specific tasks.

Programming languages come in three general forms:

Machine language:

  1. Written in numbers; the only language the computer really understands
  2. Platform specific. Machine language cannot be used on different kinds of computers.
  3. The fastest language to process

Assembly language:

  1. Uses the same instructions as in Machine Language, but keywords and variables have names rather than numbers only, making it easier to read and write.
  2. Like machine language, it is platform specific
  3. Assembly language must be translated into the machine language of a CPU by a program called an assembler.
  4. Very fast language to process

High Level Language:

  1. The closest to human language, it is easier to read, write and maintain.
  2. Platform independent: it can be used on any platform.
  3. Must be put through an interpreter or a compiler that translates the human-like language into the machine language of numbers.
  4. Slower language to process

See webopedia for more information.

Python is a High Level Program with advantages:

Python is easy to use:

It's the closest to human language, so easy to use that it's been called "programming at the speed of thought." Python programs are shorter and take less time to create than programs in many other popular languages.

Python is a "Glue" Language:

Python can be integrated with other languages such as C, C++ and Java. This means that a programmer can take advatage of work already done in another language while using Python. It also means that the strengths of other languages, such as the extra speed that C or C++ might offer.

Python has a strong community:

Python has the "Python Tutor mailing list, a more informal way for beginning programmers to ask questions The list is at: Python Tutor.

Python is also an interpreter:

It translates our code into machine language and that code can be run on any platform.

Why don't we write programs in human languages?

Natural (human) Languages are:

  • Ambiguous: Words and sentences are open to interpretation. (Example: what is the meaning of bi-weekly -- twice a week, or once every two weeks?).
  • Verbose: Natural languages use too many words and code made with them would be too bulky and slow to process.

How do programming languages work?

Like natural (human) languages, programming languages require Grammar:

Grammar:

A set of production rules that describe how to form strings from the language's alphabet that are valid according to the language's syntax.

  • String: A sequence of characters
  • Syntax: The combinations of symbols that are considered to be a correctly structured document or fragment in that language.

See the following Wikipedia pages for more information: Formal Grammar

How is human, or natural, language turned into programming language?

Using Replacement Grammars, or Context-Free Grammars, such as the Backus-Naur Form.

Replacement Grammars:

A notation used to describe each part of a language using agreed upon rules. A rule might look like the following:

  • <Sentence> ::= <subject> <verb> | <subject> <verb> <object>
  • .
  • (Which would be translated as: "A sentence consists of a subject and a verb OR a subject, verb and an object.")

In a sequence of steps, or derivations, each non-terminal (the words in < >) are gradually replaced by a terminal (such as <verb > replaced with the terminal "jump")

Replacement Grammars allow us to describe an infinitely large language with a small set of rules.

Languages made with Replacement Grammars are extremely precise!

Languages like Python, made with Replacement Grammars, are far more strict than natural languages. Rules must be followed exactly, or the instructions will not be understood or carried out.

For instance, when writing mathematical expressions in Python, you cannot mistype any part of the expression.

  • "2 + 2 _ 3" might be understood by another person as a typo for "2 + 2 + 3", but the Python interpreter will see it as an error.

Lesson 2.2: Variables and Strings

What is a variable?

In computer programming the word variable denotes a storage location containing some value that is paired with an identifier (a symbolic name). When a value has been paired with an identifier, we say that it has been assigned.

How is a value assigned?

In Python, the syntax for assigning a value to an identifier is as follows:

  • name = expression (where name is the identifier and expression is the value.)

The operator "=" is used in Python only to assign a value to an identifier. The "=" is not interpreted as "is equal to" as in standard math equations such as "1 + 1 = 2." Instead, Python uses the "==" in expressions that are meant to be read as "is equal to."

For example:

  • myName = "smebird" (assigning a value)
  • myName == "smebird" (declaring myName to be equal to the value "smebird")

Why use variables?

Variables are flexible.

  1. Variables can be either text characters ("strings,") or numbers.
  2. Changing the value or expression of a variable in one place changes it everywhere in your document.
  3. The value or expression of a variable can be as small as one character, or as long as the text of an entire book.
  4. Number values can be acted upon using mathematical operators, such as "+" (addition), "-" (subtraction), "*" (multiplication), and "/" (division).
  5. String values can be combined using the "+" symbol. This is referred to as "Concatenation," and while it uses the same symbol as addition, the effect on numbers and strings is very different.
  • For Example:
  • myString = "2" + "2" (produces the value "22," a new string)
  • myNumber = 2 + 2 (produces the value 4, an integer)

In Python, any value/expression that is enclosed in quotation marks, whether single or double, will be interpreted as a string, as in the following examples:

  • myName = "smebird"
  • someNumbers = '0123456789'
  • text_plusNumbers = '0123456789 and "smebird"'

To use Python to do any calculations, numbers are entered as a value with no quotation marks, as in the following examples:

  • myAddition = 1 + 2 + 3000000000000
  • myMultiplication = 1234 * 4321
  • myExponents = 3**9

The convention for identifiers is to start with a lowercase character. You cannot begin an identifier with a number.

Python will print strings in a different color, making it is easy to catch errors, like a forgotton end quote.

A note about changing an assigned value:

Changing the assigned value does not actually delete the former value from storage. It continues to exist, but is no longer paired to its original identifier.

If unpaired values are left in storage, programs can suffer "memory leaks" as storage reaches capacity, eventually leading to a program crash. There are Garbage Collection Modules available for Python that can be used to clear cluttered storage.

Indexing:

Another powerful way to alter a string value is using its index number to select a character or a series of characters.

Each character in a string has an assigned place value, starting with zero.

The Python syntax for selecting a particular index postion is:

  • myExample[0] (select the first character in the string)
  • myExample[some number] (select a character found in the stated position in the string)
  • myExample[0:] (select a new string starting with the first character and continuing until the last)
  • myExample[some number:some other number] (select a new string starting with the stated index number and continuing until one position before the number after the colon)
  • myExample[some number:identical number] (designates an empty string)

But, watch out for the empty string!

In the case where s = '', indexing can get strange.

Check out the following:

  • s = ''
  • s[0] + s[1:] (will output an error, because there are no characters at position 0 in the string s)
  • s[0:] (will be OK, because the colol, or subsequence operator recognizes this syntax as indicating empty string). "Why is that the case? Because!"" (Quoted from Udacity Work Session 2 video)

Using the "find" method with strings

In Python, there is a method that will allow you to find the location of any desired string. The method is "find."

The Syntax for the "find" method is: myVariable.find("desired string")

  • For example:
  • typing myVariable.find("Explain this text") might output the index number 8.

To test if the output number is correct, use it as the index number in a print command: print myVariable[8:]

  • This should output the string you were seeking: "Explain this text"

Using a second parameter, you can tell Python to locate a string past its first occurence.

The Syntax for two parameters with the "find" method is: myVariable.find("desired string", index number).

  • For example:
  • typing myVariable.find("Explain this text", 5) might output the index number 17. Indicating that the first occurence of the string, starting from index number 5, is at index number 17.

lesson2.3: Functions (or Procedures)

What is a function?

Functions are like mini-programs within your program.

They take an input (or parameter), apply a procedure to that input, and return the result -- the output.

How do we make a function?

Syntax for a function:

  • def square(n):
    • return n * n
  • print square(5)

(Output will be 25)

Vocabulary

Functions can have have one parameter, or multiple parameters separated by commas.

Each parameter takes a value as its input (called operands or arguments).

Each function must return a value derived from the code it contains. This is the output.

How do we use a function?

To use a function, we can either call it directly, for instance, by using the print command: "print square(5)," or we can insert it at part of the code in another function. This is called procedure composition, or a nested function.

When should we use a function?

Any time you have lines of code that could be used in other areas of your program. This is one of the best methods of avoiding unnecessary repetition.

Why are functions so valuable?

Procedure composition allows for powerful and flexible programming. One function can be used as a component of many other functions, reducing the amount of code that has to be written, and increasing the variety of outputs possible.

lesson2.4: Control Flow & Loops: If and While

Conditional Expressions:

The output of condtional expressions, or Boolean Conditions is always either True or False

Boolean conditions compare one or more values, using the comparison operators: "==", "!=", "<", ">".

The Boolean Operators and, or, and not are used to combine conditional expressions.

Why use conditional expressions?

Conditional expressions allow us to change a program's behavior depending on the input it receives.

Using the "or" operator to combine conditional expressions

In the following example:

  • def friends(name):
    • return name[0] == 'D' or name[0] == 'N'

We check whether our input name begins with either a 'D' or an 'N' in one line, simplifying our code.

Using "if" and "else"

Using "if" and "else" allows for additional decisions to be made within a single function.

  • def biggest(a, b, c):
    • if a > b:
      • if a > c:
        • return a
      • else:
        • return c
    • else:
      • if b > c:
        • return b
      • else:
        • return c

Using While Loops:

While loops repeatedly executes the body of the "loop" until the test condition is no longer True.

Loops are useful whenever a value needs to change in a set increment.

While loops use the comparison operators: "==", "!=", "<", ">", to test whether a test expression is True. If the result is "True", the code contained within the while loop will be executed. Once the test expression gives a value of "False", the code will be skipped and the function will drop to the next line.

Example:

  • i = 0
  • while i < 0:
    • print i
    • i = i + 1

Using Break:

Break gives us a way to stop a loop even when its test condition is True.

This gives us the means to avoid "infinite loops" that will run until the computer memory is exhausted (too long these days).

Example:

  • i = 1:
  • while i != 10:
    • if i == 11:
      • break
    • print i
    • i = i + 2

lesson2.5: Debugging

There are three main categories of bugs:

Much of the following is from "Python for Informatics," by Charles Severance. I began his Coursera course "Programming for Everybody (Python)"" in the first week of June, 2015.

Syntax errors:

Syntax errors are the easiest to make and the easiest to fix. They occur when the programmer has violated the "grammar" rules of Python. An example of a syntax error is:

  • Print 1 + 2 (In Python, the print command begins with a lowercase "p".)

Python recognizes syntax errors and will give an TraceBack error message stating the problem and the line at which the problem occurs.

Logic errors:

Logic erros occur the a program has good syntax, but there is a mistake in the order of the statements or perhaps a mistake in how the statements relate to one another. The program may not crash, but the output is unintended. A good example of a logic error might be "take a drink from your water bottle, put it in your backpack, walk to the library, and then put the top back on the bottle."

In programming, this might look like:

  • def count():
    • x = 0
    • while x < 10:
      • print x
      • x = x + 1
  • count()

Which will print the numbers from 0 to 9, instead of the intended 1 to 10.

Semantic errors:

Semantic errors can be the hardest to find. In this case, the syntax is good, the logic of the operations is correct, but the meaning of one or more lines of code is not what the programmer intended. In English, the equivalent would be like the following (copied from: Python Tutor).

  • "Children make nutritious snacks."
  • "Cocaine users are turning to ice."
  • "The building workers are refusing to work after fatal accidents."

In programming, this would be the equivalent of using an "alist[1:2]" instead of an "alist[1:3]" in an expression. The Python interpreter will have no problem executing the command, but the values returned may be incorrect.

Steps to debugging a program:

Use the Scientific Method:

  1. Observe what is happening in your program by checking the state of the program in various places.
  2. Form a hypothesis as to what your program is actually doing (as compared to what you want it to do).
  3. Make and run tests to confirm or reject your hypothesis; fix your program if you now know what is causing the problem, or observe more of your program states if you do not.

Examine error messages when programs crash.

The last line of Python Tracebacks will tell you what went wrong. Reading backwards from there will tell you more about where the problem occurred.

Work from example code.

If your modified code doesn't work, comment it out and do step-by-step modifications to the example code until it does what you want.

Make sure examples work.

Just because you find example code doesn't mean it will work in your system. Check the example code you're using to make sure it bahaves the way you expect.

Check (print) intermediate results.

When you have a working version of your code, save it before you add to the code. This will give you something to go back to if you introduce too many new bugs.

Test often!

Write the smallest testable code you can manage for each step. It's good practice both for minimizing debugging time and for reusing code later!

lesson2.6: Data Lists and For Loops

Data lists:

Data lists (like strings) are a form of structured data.

How do we create a data list?

Data lists are contained between square brackets "[]" and assigned to a variable.

A list can be empty or contain hundreds of elements (expressions). A list can contain strings, numbers or other lists.

Each element of a list must be seaparated by a comma.

What makes a list more powerful than strings?:

There are no limitations on what a list can contain. A single list can be made up of a combination of strings, integers, floats or more lists. Each element of a list has its own index number (starting at zero) and can be printed, replaced, added to or deleted. Lists contained in lists are called nested lists.

The following is an example of a nested list:

  • beatles = [['John', 1940], ['Paul', 1942], ['George', 1943], ['Ringo', 1940]]

This list gives us four elements, each of which is a list containing two pieces of information. In order to access a specific element, we call its index number.

  • print beatles[3] --- (gives us ['Ringo', 1940])

If we want to access a specific element in that nested list, we use to separate brackets.

  • print beatles[3][0] --- (gives us 'Ringo')

How do we use lists?

Two powerful concepts of lists are mutation and aliasing.

Mutation

Mutation allows us to change the value of an element. With strings, each time we assigned a new value to an existing string, we left the previous value in memory. The original value was not overridden, but was no longer paired to our variable and could not be accessed unless re-assigned.

With mutation, if we change an element, we replace the original value. This means that any variable referring to that list will be changed as well.

  • p = ['H', 'e', 'l', 'l', 'o']
  • q = p
  • p[0] = 'Y'
  • print p --- (gives us 'Yello')
  • print q --- (gives us 'Yello')

Aliasing:

When two variables, as in the above example, refer to the same objects, we call this aliasing. Aliasing can be useful when you want to enact changes across multiple variables, even when that change occurs as part of a function. This can lead to confusion if you forget that you have aliases and see a variable change in your program, and do not know why that change happened.

However, any alias can be broken by simply assigning one of the aliased variables a new value:

  • p = ['H', 'e', 'l', 'l', 'o']
  • q = p
  • p = [1, 2, 3]
  • print p --- (gives us [1, 2, 3])
  • print q --- (gives us 'Hello')

In the above example, we changed p's assigned value, just as we would do if p had been a string. A re-assignment is not the same as a mutation. Now, p and q are assigned different values and a mutation to one will not effect the other.

List operations:

Data lists have special and powerful operations.

Append

Append is like a function, but it is a method, so we use it in a way that is similar to how we use find on strings. The sytax is:

  • list.append(element)

Append adds a new element to the end of an existing list (even if it is an empty list).

  • stooges = ['Moe', 'Larry', 'Curly']
  • stooges.append('Shemp')
  • print stooges --- Outputs ['Moe', 'Larry', 'Curly,'Shemp']

Append is a mutation, so it affects all aliases. It very important to understand that if append is used to add a list containing several elements to another list, the added will be read as one element, regardless of what it contains:

  • stooges = ['Moe', 'Larry', 'Curly']
  • looney = ['Shemp', 'Marvin the Martian']
  • stooges.append(looney)
  • print stooges --- Outputs ['Moe', 'Larry', 'Curly, ['Shemp', 'Marvin the Martian'] Notice, one, and only one added element!

Also, if we change one of the items in "looney," this will change "stooges" as well:

  • looney[0] = 'Yosemite Sam'
  • print stooges --- Outputs ['Moe', 'Larry', 'Curly, ['Yosemeti Sam', 'Marvin the Martian']

List + List

  • stooges1 = ['Moe', 'Larry']
  • stooge2 = ['Curly', 'Shemp']
  • stooges1 = stooges1 + stooges2 -----(outputs ['Moe', 'Larry', 'Curly, 'Shemp'])

But does so without changing aliases, so if...

  • stooges1 = ['Moe', 'Larry']

Notice the difference between how list+list and append work! Don't confuse them!

Len operator

Len (length) looks like a function call. We pass into len the object whose length we want to know. Len works for any object that is a collection, such as strings (a collection of text characters).

Len always counts the outmost elements in a list. It does not count nested elements as separate.

  • len(['a', ['b', ['c']]]) --- (will output 2, since the outermost list contains 2 nested elements)

In the case of a string, len will output the number of characters that string contains.

Index operator

Syntax = list.index(value)

Index scans the elements in a list and outputs the index number of the first element that matches its required value.

If the desired value does not appear in the list, index outputs an error message.

In

Syntax = (value) in (list)

Index scans the elements in a list and outputs either True, if the value exists, or False, if the value is not found.

Not in

Syntax = (value) not in (list)

Index scans the elements in a list and outputs either True, if the value does not exist, or False, if the value is found.

Looping through Data Lists Using "For":

For loops are an easier way to work with lists than the previous "While" loop that we learned.

How do we create a for loop?

  • for name in list:
    • block

"Name" represents a variable that will be assigned each item in our list. Each item will be passed through the code contained in the block, and the loop will end when the last item is finished. The code found immediately after the for loop will then be evaluated.

Understand what the problem is:

You can't write code if you don't understand the problem you are trying to solve.

First, ask yourself:

  • "What are the inputs?"
  • "What are the ouputs?"

Next, ask how to get from the input to the output in a simple, mechanical way.

Break down the problem into its smallest possible components:

Start by writing your ideas as pseudocode, putting into words the steps that will eventually become code. Planning on paper before trying to work out the correct code can save enormous amounts of time.

Once you have a good idea of what steps you need to take, write small bits of code, test them and know what they do.
